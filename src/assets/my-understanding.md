แบ่ง UI ออกเป็นหลาย Component เพราะแต่ละส่วนทำหน้าที่ต่างกัน เช่น Navbar ไว้นำทาง, MemberTable ไว้แสดงตาราง, MemberForm ไว้กรอกข้อมูล และ HomePage เป็นตัวจัดการ state ทั้งหมด การแบ่งแบบนี้ทำให้โค้ดอ่านง่ายและแก้ไขได้สะดวกขึ้น
สร้าง state 3 ตัวใน HomePage ได้แก่ activeTab สำหรับเก็บว่ากดปุ่มไหนอยู่, members สำหรับเก็บรายชื่อจาก API, และ form สำหรับเก็บค่าที่พิมพ์ในช่อง input เลือกเก็บ state ไว้ที่ HomePage เพราะเป็นตัวกลางที่ส่งข้อมูลลงไปให้ Component ลูกผ่าน Props
เลือกใช้ Props แทน Context เพราะแอปนี้เล็ก state ส่งลงแค่ชั้นเดียว ไม่ต้องใช้ Context ที่ซับซ้อนกว่า
ใช้ useEffect กับ [] ว่างเพื่อ fetch ข้อมูลจาก API ตอนโหลดหน้าครั้งแรกครั้งเดียว ส่วน POST และ DELETE ไม่ต้องใช้ useEffect เพราะเรียกตอนกดปุ่มได้เลย
fetch ต้องเป็น async/await เสมอ เพราะการดึงข้อมูลจาก API ต้องใช้เวลา ถ้าไม่ใช้ async เบราว์เซอร์จะหยุดรอและผู้ใช้กดอะไรไม่ได้
